Wow, that was ugly. Now I have this:



Here’s what I learned this past month:
  • HTML (some html5) and CSS (some css3)
  • how to push updates to Github using Terminal
  • Stackoverflow is the holy grail of Q&A for progamming
  • CSSdesk is great for sandboxing... except when you accidentally go back and lose everything
  • touched on responsive and adaptive design elements (ems, percents, and @media queries)
  • @media queries for mobile compatibility though I'm still having trouble with Android devices
  • normalizing = good
  • IE = bad
  • how to install and use non-web standard fonts via fontsquirrel and googlefonts
  • dash.generalassemb.ly was a fantastic way of learning how to build a website and understand responsive design.
  • big pictures & multiple server calls = slow load times; my site should be optimized for speed.
Element-specific takeaways:
  • Background-size: cover – a big background image across an entire page looks nice and captures the full attention of the user.
  • A clean and simple nav bar at the top is key and aesthetically pleasing
  • For nav bar: tried using divs at first but it's easier to use ul & li and display: inline
  • Using border: thin solid [color] helps to visualize the space divs occupy and how margins & paddings affect them; great for nested divs and positioning
  • Differences b/w absolute, relative, and fixed positioning
  • top/right/bottom/left positioning requires position: xyz first
  • rgb is best practice (supposedly) and rgba includes opacity but not universally adopted yet
  • setting body margin & padding to 0 is critical
  • box-shadow is a neat feature; so are gradients
  • box-sizing: border-box = sweet lordy fix for dispelling margin/padding screwiness
I'm pleased with my progress so far. It makes a world of a difference building something myself; simply reading about things is not enough. The only way for me to learn is by doing.
